在C和C++等编程语言中，当尝试将一个超出带符号整数类型（如int、long等）表示范围的值赋给该类型的变量时，结果是未定义的（Undefined Behavior，UB）。这意味着程序可能以任何方式表现，包括但不限于：

  1. 数据截断：在某些情况下，超出范围的值可能会被截断以适应目标类型的范围。但是，这种截断是未定义的，因为不同的编译器或平台可能会以不同的方式实现它。

  2. 数据溢出：值可能会溢出到类型的另一端。例如，对于int类型（假设为32位），如果尝试赋值为2147483648（即2^31，超出了int的最大正值2147483647），则结果可能是-2147483648（即int的最小负值）。但是，这种溢出行为也是未定义的，因为编译器可能会选择以其他方式处理这种情况。

  3. 程序崩溃：在某些情况下，尝试执行未定义行为可能会导致程序崩溃或产生不可预测的结果。这取决于编译器的实现、目标平台的硬件特性以及程序的其他部分。

  4. 安全漏洞：未定义行为还可能导致安全漏洞，如缓冲区溢出攻击，其中攻击者通过精心构造的输入来触发未定义行为，从而利用程序中的漏洞。

因此，编写涉及带符号整数类型的代码时，必须非常小心以避免溢出和未定义行为。这通常包括：

  1. 使用更大的数据类型（如long long）来存储可能超出较小类型范围的值。
  2. 在赋值之前进行范围检查，以确保值在目标类型的有效范围内。
  3. 使用库函数（如C标准库中的atoi的替代品strtol，它提供了错误检查）来处理从字符串转换到整数的操作。
  4. 避免在算术运算中混合使用带符号和无符号整数类型，因为这可能导致意外的溢出和未定义行为。
  • 程序尽量避免依赖于实现环境的行为，比如int的尺寸在不同环境可能不同。

4 初始化

初始化不是赋值，初始化是创建变量时赋予一个初始值，赋值是把对象的当前值擦除并用一个新值来替代。

花括号，圆括号以及":"的区别

  • 使用{}进行初始化的方法被称为列表初始化（list initialization），这是在C++11被引入的特性；
  • 而使用（）进行初始化的方法是传统的初始化语法，可以用来初始化变量，也可以用来调用函数;
  • 在构造函数参数列表后面使用冒号“:”并以逗号分隔的列表形式来初始化成员变量的方法被称为初始化列表（member initializer list），其实这里的初始化列表应该翻译成成员初始化列表，初始化列表是在类的构造函数中使用的语法，用于在构造函数体执行之前初始化类的成员变量，初始化列表可以在构造函数体之前为成员变量赋初值，从而提供了更高效的初始化方式。

列表初始化相较于传统初始化的区别

当用于内置类型的变量时，使用列表初始化且初始值存在信息丢失的风险，编译器会报错。

  • 定义于函数体内的内置类型的对象如果没有初始化，则其值未定义，定义于任何函数之外的内置类型则被初始化为0；
  • 类的对象如果没有显式的初始化，则其由类确定。string默认初始化为一个空串。
  • 不能使用未初始化的变量，否则会引发运行时故障。

声明

  • 声明和定义是严格区分的。要声明一个变量需要加extern，声明变量不能赋值。任何包含了显式初始化的声明即成为定义：
  • 变量只能被定义一次，但是可以多次声明。
  • C++是静态类型语言，在编译阶段检查类型。

8 处理类型

类型别名

有两种方法定义类型别名：

typedf作为声明语句中的基本数据类型的一部分出现，含有typedef的声明语句定义的不再是变量而是类型别名，和其他声明语句一样，typedef的声明语句中也可以包含类型修饰符，从而构造符合类型。

这里的关键在于理解 const 修饰的是指针本身还是指针指向的数据。在 const pstring 的情况下，const 修饰的是指针（即 cstr），而不是 cstr 所指向的 char 类型数据。这与 const char* 不同，后者表示一个指向 const char 的指针，意味着你不能通过这个指针修改它所指向的数据，但你可以让指针指向其他数据。
因此，注释中提到的“不能采用直接替换的方式将其理解为 const char* cstr = 0”是正确的。虽然两者看起来相似，但它们的行为是不同的：

  • const pstring cstr = 0; 声明了一个指向 char 的常量指针 cstr，并初始化为空指针。你不能改变 cstr 指向其他地址，但可以（如果它指向有效内存的话）通过其他方式修改它所指向的数据。
  • const char* cstr = 0; 声明了一个指向 const char 的指针 cstr，并初始化为空指针。这意味着你不能改变 cstr 指向的数据，但你可以让 cstr 指向其他数据（只要那些数据也是 const char 类型）。

auto类型说明符

  • auto说明符让编译器根据初始值来分析表达式所属的类型，使用auto会增加编译时间，但不会增加运行时间。
  • auto可以在一条语句中声明多个变量，但是多个变量必须是同一个基本数据类型（整型于整型指针和整型引用算一个类型）

编译器推断出的auto类型有时和初始值并不一样，编译器会进行适当的调整：

  1. auto根据引用来推断类型时会以引用对象的类型作为auto的类型。
  2. auto一般会忽略掉顶层const，因此对于非指针类型的常量对象，auto推断出的结果是不含const的，如果希望auto是一个顶层const，需要明确指出。
  3. auto会保留底层const。

概况一下下就是auto会忽略引用与顶层const。

我们着重分析一下第5行和第六行的代码：

  • auto &e = ci;
    这行代码是错误的。如之前所述，非常量引用不能绑定到常量对象上。因此，尝试创建一个对 ci 的非常量引用 e 是不允许的，并且会导致编译错误。
  • auto f = &ci;
    这行代码是正确的，但注释有误。f 的类型被推导为 int* const 的简化形式，即 int*（因为 auto 推导时不会保留指针所指向对象的常量性，除非使用 decltype 或类似的机制）。然而，这里的重点是 f 存储的是 ci 的地址，而不是 ci 的值。f 是一个指向 int 的指针，而不是一个 const int*（尽管它指向的对象是常量）。这里的“底层 const”概念并不适用于指针类型本身，而是适用于指针所指向的对象。在这个情况下，由于 ci 是常量，你不能通过 f 来修改它所指向的值，但 f 本身（即指针）是可以修改的（可以指向另一个地址）。然而，如果你想要表达 f 指向一个常量 int，你应该显式地声明为 const int* f = &ci;。
  • iint与int*、int&是一个基本数据类型，而const int与int不是一种类型。
  • 用auto定义引用时，必须用&指明要定义的是引用。

  • 如果decltype使用的表达式是一个变量，则它返回该变量的类型（包括顶层const和引用在内）。

  • decltype与auto的不同，decltype不会忽略引用和顶层const。

  • 注意当获得的类型是引用时，必须初始化。

  • 引用从来都是作为对象的别名出现，只有在decltype处是例外。

  • 如果decltype使用的表达式不是一个变量，则decltype返回表达式结果对应的类型。可以使用这种方式来保证不获取引用类型。

  • 注意解引用指针的结果是一个引用类型，给变量加括号的结果也是引用类型，赋值操作的结果也是引用类型。

    10 自定义数据类型

    • 类通常定义在头文件中，类所在头文件的名字应与类的名字一样。
    • 头文件通常定义那些只能被定义一次的实体，比如类、const、constexpr等。
    • 头文件一旦改变，相关的源文件必须重新编译以获取更新过的声明。
    • 确保头文件多次包含仍能安全工作的常用技术是预处理器。
    • 预处理变量有两种状态：已定义和未定义。一般把预处理变量的名字全部大写。
    • 整个程序中的预处理变量包括头文件保护符必须唯一，通常基于头文件中类的名字来构建保护符的名字，以确保其唯一性，C++中包含下面这三个头文件保护符：
      1.#define：把一个名字设定为预处理变量。
      2.#idndef当且仅当变量已定义时为真，一旦检查结果为真，则执行后续操作直到遇到#endif为止。
      3.#endif

  • 预处理变量无视作用域的规则，作用范围是文件内。

11 一些问题

  • 指针和引用有4点不同，分别是哪些？
  • 指针是对象而引用不是；指针可以重定向引用不可以；有指向指针的指针无引用的引用；引用必须初始化而指针不需要。
  • const对象必须怎样？
  • 必须初始化。
  • const对象的作用范围？
  • 默认范围是文件内。
  • 什么是常量引用、如何声明、最顶层还是最底层？
  • 不能改变对象的引用是常量引用，const int& i = a，是底层const。
  • 常量指针与常量对象、非常量对象的关系？
  • 不能用非常量引用绑定常量对象，可以用常量引用绑定非常量对象。
  • 什么是常量指针，如何声明，是顶层还是底层？
  • 常量指针表明指针是个常量，其内存储的地址不能改变，但是指针还能修改所指对象的值，int* const p = a，是顶层const。
  • 常量指针与常量对象、非常量对象的关系？
  • 可以用常量指针指向非常量对象。
  • 顶层const和底层const都是什么，在什么位置？
  • 只有指针同时有顶层和底层，const在星号右边是顶层，左边是底层；引用的const是底层；其他类型const是顶层。
  • constexpr是什么、特点是什么？
  • 常量表达式。两个点，值不能改变，在编译阶段就可以计算出值。
  • 浮点数赋给整型变量时如何舍入？
  • 只保留小数点前的部分，即向零舍入。
  • decltype是什么、如何使用？
  • 用来获取变量类型；decltype a。
  • 如何声明而非定义一个变量？
  • 使用extern修饰符；extern int i；
  • 如果指针不初始化会有什么影响？
  • 在块作用域中，未初始化的指针的值是未定义的。
  • 如何在多个文件间共享const对象？
  • 如果想在多个文件间共享const对象，必须在变量的定义前添加extern关键字并在本文件中声明，声明和定义都要加extern。
  • 使用auto来定义引用时要注意什么？
  • 用auto定义引用时，必须要加&符号，尤其是在范围for循环中，当想要修改值时，一定要记得加上引用符。
  • 预处理变量的作用范围是什么？
  • 文件内。
  • C++属于静态型语言，静态类型语言的含义是什么？
  • 静态类型语言在编译时检查变量类型。
  • C++有两种定义类型别名的方式，分别都是什么？
  • typedef unsigned int size_tyoe和using size_type = unsigned int;

这里需要解释一下typedef和using定义类型别名的区别：

  • 基本语法与功能

  1. typedef：

    • 语法：typedef 原类型 别名;
    • 功能：为已存在的类型提供一个新名称（同义词）。
    • 示例：typedef int integer;，这里integer成为了int的别名。

  2. using：

    • 语法：using 别名 = 原类型;
    • 功能：同样为已存在的类型提供一个新名称，并且可以用于模板别名。
    • 示例：using integer = int;，这里integer也成为了int的别名。
  • 模板别名的支持
  1. typedef：不支持模板别名。如果需要为模板类型定义别名，通常需要使用结构体（struct）或其他技巧来间接实现。
  2. using：支持模板别名。这使得在使用模板时，可以更加简洁和直观地定义类型别名。
    • 示例：template<typename T> using MyVector = std::vector<T>;，这里MyVector成为了std::vector<T>的模板别名。
  • 命名空间的别名
  1. typedef：不能用于命名空间的别名定义。
  2. using：可以用于命名空间的别名定义，从而简化对命名空间中成员的访问。
    • 示例：namespace std_alias = std;，然后使用std_alias::cout来访问std::cout。
  • 依赖名称查找与多重定义
  1. typedef：在依赖名称查找时，如果存在多个同名的别名，可能不会报错，但可能会导致歧义。
  2. using：在依赖名称查找时，如果存在多个同名的别名，编译器会报错，从而避免潜在的歧义。

在现代C++（C++11及以后版本）中，由于using提供了更强大的功能和更清晰的语法，因此通常建议优先使用using而非typedef来定义类型别名。
