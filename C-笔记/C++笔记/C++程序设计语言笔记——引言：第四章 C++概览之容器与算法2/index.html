0 容器

如果一个类的主要目的是保存一些对象，那么我们通常称之为容器。
C++ 标准模板库（STL）提供了多种容器，这些容器可以分为三大类别：序列容器、关联容器和容器适配器。每种类型的容器都有其特定的用途和特点，选择合适的容器对于编写高效且易于维护的代码至关重要。

1. 序列容器

序列容器按照插入顺序存储元素，并提供连续访问元素的功能。这类容器的特点是它们内部维护着元素之间的线性关系，即每个元素都有一个明确的位置，这个位置是由元素插入的时间点决定的，而不是由元素本身的值决定的。

  • vector：动态数组，支持随机访问，大小可变。它在内存中以连续的方式存储元素，因此非常适合需要频繁随机访问但较少进行插入或删除操作的场景。

  • deque（双端队列）：支持在两端进行快速插入/删除。虽然也允许随机访问，但在中间位置插入或删除元素时效率较低。

  • list：双向链表，支持在开头和结尾快速插入/删除元素。由于元素不是连续存储的，所以不支持高效的随机访问。

  • forward_list（C++11新增）：单向链表，只支持向前遍历。相比 list 更加轻量级，适用于只需要单向遍历的情况。

  • array（C++11新增）：固定长度的数组，提供了更好的类型安全性和边界检查，但一旦创建后其大小不可改变。

2. 关联容器

关联容器根据元素的关键字来组织数据，通常用于需要快速查找、插入或删除操作的场景。它们内部使用平衡二叉搜索树（如红黑树）或哈希表实现，确保了对数级别的复杂度或者平均情况下常数级别的性能。

  • set 和 multiset：分别表示不允许重复和允许重复的有序集合。所有元素都是唯一的，并且按照一定规则排序。

  • map 和 multimap：键值对映射，基于红黑树实现。map 中的键是唯一的，而 multimap 允许多个相同的键存在。

  • unordered_set 和 unordered_multiset（C++11新增）：无序集合，基于哈希表实现，不允许重复元素。提供了比有序集合更快的查找速度，但在最坏情况下可能会退化为线性时间复杂度。

  • unordered_map 和 unordered_multimap（C++11新增）：无序键值对映射，同样基于哈希表实现。与 unordered_set 类似，提供了更快的查找速度。

3. 容器适配器

容器适配器并不直接管理元素，而是通过包装其他容器来提供特定接口。它们通常用来实现栈、队列等抽象数据类型。

  • stack：后进先出（LIFO）的数据结构。默认底层容器为 vector，也可以指定为 deque 或 list。

  • queue：先进先出（FIFO）的数据结构。默认底层容器为 deque，也可以指定为 list。

  • priority_queue：优先级队列，总是将最大（或最小）元素放在队首。默认底层容器为 vector，也可以指定为 deque 或 list。

其中要额外说明的是，在C++23中引入了平坦容器，“平坦容器”指的是那些试图通过使用连续内存存储（如数组）来优化某些操作性能的容器。与传统的基于树或哈希表的关联容器不同，平坦容器利用了连续内存的优点，比如更好的缓存局部性和更快的遍历速度。然而，这种设计通常会牺牲一些插入和删除操作的效率，因为这些操作可能需要移动大量数据以维持顺序。

平坦容器的种类

在C++23标准提案中，讨论了几种新的平坦容器，它们是：

  • flat_set：一个按键排序且不允许重复元素的集合。
  • flat_multiset：类似于flat_set，但是允许重复元素。
  • flat_map：一个按键排序的键值对集合，其中每个键都是唯一的。
  • flat_multimap：类似于flat_map，但是允许相同的键对应多个值。

特性与优缺点

平坦容器的主要特性包括：

  • 快速查找：由于底层使用的是有序数组，查找可以采用二分查找算法，时间复杂度为O(log n)。
  • 空间紧凑：连续内存布局使得数据更加紧凑，减少了内存碎片，并提高了缓存命中率。
  • 随机访问支持：因为元素是连续存储的，所以支持高效的随机访问。

然而，平坦容器也有其局限性：

  • 插入/删除效率低：当插入或删除元素时，可能需要移动大量的元素以保持有序状态，这导致了较高的开销。
  • 迭代器失效问题：任何修改容器的操作都可能导致现有的迭代器失效。

实现细节

平坦容器通常是在现有的关联容器基础上进行改造，将原本基于红黑树的数据结构替换为基于数组的数据结构。例如，在flat_set和flat_multiset中，数据被存储在一个有序的数组里；而在flat_map和flat_multimap中，则是用两个数组分别存放键和值，或者是一个包含键值对的数组 。

vector

最有用的标准库容器当属vector，一个vector就是一个给定类型元素的序列，元素在内存中是连续存储的。拷贝和移动vector是通过构造函数和赋值运算符实现的。vector的赋值过程包括拷贝其中的元素。当一个vector包含很多元素时，这样一个看起来无害的赋值或初始化操作可能非常耗时。因此，当拷贝并非必要时，我们应该优先使用引用或指针或是移动操作。

性能考量

根据一些实验结果，std::vector 在某些场景下的性能可能不如固定大小的数组。特别是当涉及到大量的插入、删除或者随机访问操作时，由于 vector 需要频繁地调整内部缓冲区的大小，这可能会导致额外的时间消耗。此外，如果 vector 中存储的是复杂类型的对象，则每次复制这些对象也会增加额外的开销。因此，在高性能要求的应用程序中，开发者应该仔细评估是否真的需要使用 vector，或者是否有更合适的数据结构可用。

list

C++标准库提供了一个名为list的双向链表，如果我们希望在一个序列中添加和删除元素的同时无需移动其他元素，则应该使用list。当使用链表时，我们通常并不想像使用向量那样使用它，即，不会用下标操作来访问链表元素，而是想进行“在链表中搜素具有给定值的元素”这类操作。为了完成这样的操作我们可以利用“list是序列”这样一个事实。
![在这里插入图片描述](./C++程序设计语言笔记——引言：第四章 C++概览之容器与算法2image\edae3d740fae471cbaf2cea43b6a5ccf.png)
C++ 标准库中的 std::list 是一个双向链表容器，它允许在序列中任何位置以常量时间复杂度 O(1) 进行插入和删除操作，而无需移动其他元素。这使得 std::list 成为频繁执行插入和删除操作的理想选择，尤其是在需要在序列中间进行这些操作的情况下。与 std::vector 不同，std::list 的元素不是连续存储的，因此不支持通过下标直接访问元素的操作。相反，std::list 提供了迭代器来遍历和操作元素，这使得我们可以更灵活地处理数据。

在链表中搜索具有给定值的元素

由于 std::list 不支持随机访问，我们不能像使用 std::vector 那样通过下标来快速定位元素。然而，std::list 作为一个序列容器，提供了多种方法来查找特定值的元素。以下是几种常见的搜索方式：

使用标准算法 std::find

std::find 是 C++ 标准库提供的一个通用算法，它可以用于任何支持迭代器的容器，包括 std::list。std::find 会从给定的起始迭代器开始，逐个检查每个元素，直到找到与给定值匹配的元素或到达结束迭代器为止。如果找到了匹配的元素，则返回指向该元素的迭代器；否则返回结束迭代器 end()。

使用成员函数 remove 和 remove_if

虽然 remove 和 remove_if 主要用于删除元素，但它们也可以用来查找符合条件的元素。remove 会移除所有等于指定值的元素，而 remove_if 则根据自定义的谓词移除元素。需要注意的是，remove 和 remove_if 并不会改变容器的大小，而是将被移除的元素移到容器的末尾，并返回一个新的结束迭代器。因此，如果我们只想查找而不删除，可以结合 std::find 或 std::find_if 来实现。

使用成员函数 find（C++20）

从 C++20 开始，std::list 引入了成员函数 find，可以直接用于查找特定值的元素。find 返回指向找到的第一个匹配元素的迭代器，如果没有找到则返回 end()。这简化了查找操作，使得代码更加简洁。

利用 std::list 的特性

除了上述的查找方法外，std::list 还提供了许多其他有用的成员函数和算法，可以帮助我们在链表中高效地进行各种操作。例如：

  • insert：可以在指定位置插入一个或多个元素。std::list 的 insert 函数接受一个迭代器作为插入点，并可以在该位置之前插入新元素。插入操作不会影响其他元素的位置，因此非常适合在链表中间插入元素。

  • erase：可以从链表中删除一个或多个元素。erase 接受一个迭代器或一对迭代器，表示要删除的元素范围。删除操作也不会影响其他元素的位置，因此非常适合在链表中间删除元素。

  • splice：可以将另一个 std::list 中的元素移动到当前链表的指定位置。splice 是一个非常高效的操作，因为它只是修改指针而不涉及元素的复制或移动。

  • merge：可以将两个已排序的 std::list 合并成一个有序的链表。merge 会将两个链表中的元素按顺序合并，并且不会创建新的节点，而是直接重用现有的节点。

  • sort：可以对链表中的元素进行排序。sort 是一个成员函数，可以直接用于 std::list，并且不需要额外的空间来存储临时数据。排序后，链表中的元素将按照升序排列，或者可以根据自定义的比较函数进行排序。

1 算法

无序容器针对关键字（通常是一个字符串）搜索进行了优化，这是通过使用哈希表来实现的。
前面提到的容器分别定义在 、 和 等头文件中，这些容器都属于名字空间 std。此外，标准库还提供了容器适配器 queue、stack、deque和 priority_queue。标准库还提供了一些更特殊的类似容器的类型，如定长数组 array<T,N>和 bitset。
从符号表示的角度来看，标准库的各种容器和它们的基本操作比较类似。而且，不同容器的操作含义也大致相同。基本操作可用于每一种适用的容器，且可高效实现。例如：

  • begin() 和 end() 分别返回指向首元素和尾后位置的迭代器。
  • push_back() 可用来（高效地）向 vector、list 及其他容器末尾添加元素。
  • size() 返回元素数目。

形式和语义上的一致性使得程序员可以设计出与标准库容器在使用方式上非常相似的新的容器类型，包含范围检查功能的向量 Vector就是一个很好的例子。容器接口的一致性还便于我们设计与容器类型无关的算法。但是，凡事皆有两面性，某种技术的优点和缺点常常并存。例如，下标操作和遍历 vector 的操作很高效也很简单。但另一方面，当我们在 vector 中插入或删除元素时，不得不每次都移动元素，效率不佳；list 则恰好具有相反的特性。请注意，当序列较短且元素尺寸较小时，vector 通常比 list 高效（insert() 和 erase() 操作也是如此）。推荐将标准库 vector 作为存储元素序列的默认类型；只有当你的理由足够充分时，再考虑选择其他容器。
对于编程任务来说，仅仅定义链表和向量等数据结构是远远不够的。为了使用某种数据结构，我们还需要一些基本访问操作，比如添加和删除元素的操作（就像为 list 和 vector 提供的那样）。而且，我们一般不会只把对象保存在容器中了事，而是需要对它们进行排序、打印、抽取子集、删除元素、搜索等更复杂的操作。因此，标准库在提供最常用的容器类型之外，还为这些容器提供了最常用的算法。
迭代器的一个重要作用是分离算法和容器。算法通过迭代器来处理数据，但它对存储元素的容器一无所知。反之亦然，容器对处理其元素的算法也是一无所知。反之亦然，容器对处理其元素的算法也是一无所知，它所做的全部事情就是按需求提供迭代器（如begin()和end()）。这种数据存储和算法分离的模型催生出非常通用和灵活的软件。
迭代器本质上是什么？当然，任何一种特定的迭代器都是某种类型的对象。不过，迭代器的类型非常多，毕竟每个迭代器都是与某个特定容器类型相关联的。它需要保存一些必要的信息，以便我们对容器执行某些特定的任务。因此，有多少种特殊要求就有多少种迭代器。例如，vector的迭代器可能就是一个普通指针，因为就引用vector中的元素而言指针再恰当不过了：

所有迭代器类型的语义及其操作的命名都是相似的。例如，对任何迭代器使用++运算符都会得到一个指向下一个元素的迭代器，而*运算符则得到迭代器所指的元素。实际上，任何符合这些简单规则的对象都能被看成是迭代器。用户不需要知道某个特定迭代器的类型，迭代器“知道”它自己的迭代器类型是什么，而且都能通过规范的名字iterator和const_iterator来正确声明自己的类型。例如，list::iterator是list的迭代器类型，程序员很少需要操心“这些类型是如何定义的？”等细节问题。
迭代器是处理容器中元素序列的一个很有用的通用概念。但是，容器并非容纳元素序列的唯一场所。例如，一个输入流产生一个值的序列，我们还可以将一个值的序列写入一个输出流。因此，将迭代器的概念应用到输入输出是很有用的。
在前面的例子中，算法都是对序列中每个元素简单地进行“内置”操作，但我们常常需要把操作也作为算法的参数。例如，find算法提供了一种查找给定值的便捷途径。而对于查找满足特定要求元素的问题，还有一种更通用的方法，称为谓词。例如，我们可能需要在一个map中搜索第一个大于42的值。我们在访问map的元素时，实际访问的是（关键字，值）对的序列。因此，我们可以将任务转化为在map<string,int>中搜索一个特定的pair<const string,int>，并要求它的int部分大于42。
算法的一个更一般性的定义是“一个有限规则集合，给出了一个操作序列，用来求解一组特定问题 [且]具有5个重要特性：有限性……确定性……输入……输出……有效性”。在C++标准库的语境中，算法就是一个对元素序列进行操作的函数模板。
标准库提供了很多算法，它们都定义在头文件中且属于名字空间std。这些标准库算法的输入都是序列。一个从b到e的半开序列表示为[b:e)。下面是一些特别有用的算法的简介：

调用 描述
p=find(b,e,x) p 是 [b:e) 中第一个满足 *p==x 的迭代器
p=find_if(b,e,f) p 是 [b:e) 中第一个满足 f(*p)==true 的迭代器
n=count(b,e,x) n 是 [b:e) 中满足 *q==x 的元素 *q 的数量
n=count_if(b,e,f) n 是 [b:e) 中满足 f(*q)==true 的元素 *q 的数量
replace(b,e,v,v2) 将 [b:e) 中满足 *q==v 的元素 *q 替换为 v2
replace_if(b,e,f,v2) 将 [b:e) 中满足 f(*q)==true 的元素 *q 替换为 v2
p=copy(b,e,out) 将 [b:e) 复制到 [out:p)
p=copy_if(b,e,out,f) 将 [b:e) 中满足 f(*q)==true 的元素 *q 复制到 [out:p)
p=unique_copy(b,e,out,f) 将 [b:e) 复制到 [out:p)，不复制连续的重复元素
sort(b,e) 使用 < 作为排序准则，对 [b:e) 中的元素进行排序
sort(b,e,f) 使用谓词 f 作为排序准则，对 [b:e) 中的元素进行排序
(p1,p2)=equal_range(b,e,v) [p1:p2) 是已排序序列 [b:e) 中值等于 v 的子序列，本质上相当于对 v 进行二分查找
p=merge(b,e,b2,e2,out) 将两个序列 [b:e) 和 [b2:e2) 合并，结果存储到 [out:p)

2 建议

1.没必要推倒重来，直接使用标准库是最好的选择；
2.除非万不得已，大多数时候先考虑使用标准库，再考虑别的库；
3.标准库绝非万能；
4.如果你用到了某些标准库设施，记得把头文件#include进来；
5.标准库设施位于名字空间std中；
6.标准库string优于C风格字符串（即char*）;
7.iostream具有类型敏感、类型安全和可扩展等特点；
8.与T[]相比，vector、map<K,T>和unordered_map<K,T>更优；
9.一定要了解各种标准库容器的设计思想和优缺点；
10.优先选用vector作为你的容器类型；
11.数据结构应力求小巧；
12.如果你拿不准会不会越界，记得使用带边界检查的容器（比如Vec）；
13.用push_back()或者back_inserter()给容器添加元素；
14.在vecto上使用push_back()要比数组上使用realloc()更好；
15.在main()函数中捕获常见的异常；
16.了解常用的标准库算法，用它们代替你自己手写的循环；
17.如果用迭代器实现某算法过于冗长，不妨改写成使用容器的版本。
