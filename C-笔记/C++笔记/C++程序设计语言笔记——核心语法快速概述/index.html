区分字符字面量和字符串字面量

字符串字面量即用双引号括着的多个字符字面量就是字符字面量的序列。并且字符串字面量是存在于字面量池中，字面量池中的字符串字面量是在程序初始化的时候填出入字面量池中，其生命周期直到加载该字面串字面量的程序退出。我们可以将字面量池想象成一份“签到表”，而字符串字面量就比喻成众多的签名,但这份签到表比较特殊。

字面量池的特征

字面量池就是位于C/C++中内存布局中较向低地址的一片内存区域，这片内存区域的通常是公开的，整个程序的上下文用到所有字符字面量都会在初始化过程中填入该池，并且全局维护整个程序上下文的唯一副本。也就是其他内存区域用到字符串字面量副本是从该池中“初次”(注意我的用词)拷贝过去的发源地。该池的特征如下:

  • 不允许有重复的字符串字面量出现，这样做的目的是为了节省空间,例如"hello word"不可能出现两次,但“hello world ”,注意后者的字符字面量最后是一个空格即

面向对象的核心概念

面向对象编程（Object-Oriented Programming，简称OOP）是一种编程范式或编程风格，它以“对象”作为基本单位，利用类和对象来组织代码和数据。OOP的核心思想是“万物皆对象”，它将现实世界中的实体抽象为对象，并通过类和对象来实现数据的封装、继承和多态等特性。
以下是面向对象编程的几个关键概念：

  • 对象（Object）：对象是类的实例，它包含数据（称为属性或字段）和操作数据的方法（称为行为或函数）。对象是面向对象编程中的基本单位。
  • 类（Class）：类是对象的蓝图或模板，它定义了对象的属性和方法。通过类，我们可以创建具有相同属性和方法的多个对象。
  • 封装（Encapsulation）：封装是将数据（属性）和操作数据的方法（方法）结合在一个类中，并对外界隐藏其内部实现细节的过程。封装提高了代码的安全性和可维护性。
  • 继承（Inheritance）：继承是面向对象编程中实现代码复用的重要机制。通过继承，一个类（子类或派生类）可以继承另一个类（父类或基类）的属性和方法。这允许我们创建具有层次结构的类体系，并在其中共享和重用代码。
  • 多态（Polymorphism）：多态是面向对象编程中实现接口重用的重要机制。它允许不同的类以统一的接口相互操作。多态性通常通过继承和虚函数来实现，它使得程序能够在运行时根据对象的实际类型来选择调用哪个方法。

面向对象编程在软件开发中得到了广泛的应用，特别是在大型复杂系统的开发中。它提供了一种更加自然和直观的方式来组织和表达现实世界中的实体和关系。

虚函数

在C++中，虚函数（virtual function）是面向对象编程中的一个重要概念，它允许在派生类（子类）中重写基类（父类）中的成员函数。通过使用虚函数，可以实现多态性（polymorphism），即在运行时根据对象的实际类型来确定调用哪个函数版本，而不是在编译时根据对象的静态类型来确定。

虚函数的作用:

  • 实现多态性
    虚函数允许通过基类指针或引用来调用派生类中的函数实现。这是多态性的核心，使得基类指针可以指向派生类对象，并在运行时调用相应的派生类函数。
  • 动态绑定
    虚函数在运行时（而非编译时）确定具体调用哪个函数。这是通过虚函数表（vtable）实现的，每个包含虚函数的类都有一个虚函数表，表中存储了指向这些虚函数的指针。
  • 代码复用和扩展：
    通过虚函数，可以在基类中定义通用接口，并在派生类中提供具体实现。这不仅简化了代码，还提高了代码的可维护性和可扩展性。
  • 注意事项
    纯虚函数：在基类中可以将一个虚函数声明为纯虚函数（pure virtual function），使用= 0语法。这样，基类成为抽象基类（abstract base class），不能实例化，但可以用于定义接口。

析构函数：如果基类有虚函数，通常建议将基类的析构函数也声明为虚函数。这确保了当通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，从而避免资源泄漏。

构造器(构造函数)与析构器（析构函数）

在C++中，构造器（Constructor）是一种特殊的成员函数，其主要功能是在创建对象时初始化该对象。构造器与类同名，并且没有返回类型（包括void也不允许）。每当创建类的新实例时，构造器都会被自动调用，以确保对象在使用前被正确初始化。
构造器的主要特点和作用包括：

  • 初始化对象：构造器负责为对象的成员变量设置初始值。这是确保对象在创建时处于有效状态的关键步骤。
  • 无返回类型：与类的其他成员函数不同，构造器没有返回类型。这意味着它们不能返回任何值，包括void。
  • 与类同名：构造器的名称必须与类名完全相同，这样编译器才能在创建对象时识别并调用它。
  • 自动调用：每当使用类名创建新对象时，构造器都会自动被调用。无需显式地在代码中调用它。
  • 重载：C++允许为同一个类定义多个构造器，只要它们的参数列表不同。这称为构造器的重载，它提供了创建对象时的灵活性。
  • 析构器的对应：构造器与析构器（Destructor）是相对应的。析构器在对象生命周期结束时被调用，用于执行清理工作，如释放内存、关闭文件等。
  • 默认构造器：如果一个类没有定义任何构造器，编译器会提供一个默认的无参数构造器。但是，如果类中定义了至少一个构造器（无论是有参数的还是无参数的），编译器就不会再提供默认构造器。
  • 成员初始化列表：在构造器的函数体中初始化成员变量是一种方法，但更好的做法是使用成员初始化列表。成员初始化列表在构造器的函数体执行之前执行，并且对于某些类型（如引用和const成员）来说是必需的。

为什么要使用初始化列表？
效率：对于某些类型（如内置类型int、float等），使用初始化列表和直接在构造器函数体中赋值之间的效率差异可能不明显。但是，对于类类型（特别是那些涉及动态内存分配或复杂初始化逻辑的类），使用初始化列表可以避免不必要的复制或移动操作，从而提高效率。
必要性：对于const成员和引用成员，它们必须在初始化列表中初始化，因为一旦它们被声明为const或引用，就不能再被赋值了。
一致性：使用初始化列表可以确保所有成员变量都在构造器函数体之前被初始化，这有助于保持代码的一致性和可读性。
避免默认构造函数调用：如果类成员是另一个类的对象，并且该成员类定义了自己的构造器（特别是非默认构造器），则在初始化列表中初始化该成员可以避免调用其默认构造器，然后再进行赋值操作。这可以减少不必要的构造和析构开销。
总之，初始化列表是C++中一种强大且高效的方式来初始化类的成员变量。在可能的情况下，应该优先考虑使用初始化列表来初始化成员变量。

多态

多态（Polymorphism）是面向对象编程中的一个核心概念，它指的是同一个操作或函数调用在不同的对象上可以有不同的行为或结果。多态性允许我们将对象视为其基类类型（或接口类型）的实例，并在运行时根据实际对象的类型来决定调用哪个具体的方法实现。
多态性主要分为两种类型：

  • 编译时多态（静态多态）：这通常通过函数重载（Function Overloading）和运算符重载（Operator Overloading）来实现。在编译时，编译器根据函数的参数列表或运算符的操作数类型来确定应该调用哪个具体的函数或运算符实现。这种多态性在编译时就已经确定，因此也被称为静态多态性。
  • 运行时多态（动态多态）：这是通过继承和虚函数（Virtual Functions）来实现的。在运行时，程序根据对象的实际类型来确定应该调用哪个具体的方法实现。这种多态性在运行时才确定，因此也被称为动态多态性。

在运行时多态中，基类通常包含一个或多个虚函数，这些函数在派生类中被重写（Override）。当通过基类的指针或引用来调用这些虚函数时，C++（或其他支持多态性的面向对象语言）的运行时系统会根据对象的实际类型来确定应该调用哪个版本的函数。

子类多态

在C++中，子类多态（Subclass Polymorphism）是面向对象编程中的一个关键特性，它允许我们使用基类（或父类）的指针或引用来引用派生类（或子类）的对象。这种机制使得我们可以在不知道具体子类类型的情况下，通过基类接口来调用子类中重写的方法。
多态性主要通过以下两种方式实现：

  • 函数重载（Overloading）：这通常指的是在同一个作用域内，函数名相同但参数列表不同的函数。然而，在C++多态性的上下文中，更常见的是通过继承和虚函数来实现多态。函数重载本身并不直接提供多态性，但它是多态性实现中的一个有用工具。
  • 虚函数（Virtual Functions）：在基类中，将函数声明为virtual，允许派生类提供该函数的特定实现。当通过基类指针或引用调用虚函数时，C++运行时系统会根据对象的实际类型来确定应该调用哪个版本的函数。这是实现多态性的核心机制。

使用 override 关键字的好处包括：

  • 提高代码的可读性：它清晰地表明了这个函数是故意设计为重写的，这对于阅读和维护代码的人来说是一个有用的提示。
  • 编译时检查：如果基类中的虚函数签名发生了变化，或者你打算重写的函数实际上并没有在基类中声明为虚函数，那么使用 override 关键字的代码将无法编译。这有助于在早期发现潜在的错误。
  • 防止意外重载：有时，由于函数签名相似（比如参数类型不同但数量相同），可能会不小心创建了一个重载函数而不是重写函数。使用 override 关键字可以避免这种情况。

Derived 类中的 show() 函数后面的 override 关键字表明这个函数是故意设计为重写 Base 类中的虚函数 show() 的。如果 Base 类中没有名为 show() 的虚函数，或者 Derived::show() 的函数签名与 Base::show() 不匹配（比如参数类型或数量不同），编译器将报错。
这种机制有助于保持代码的正确性和一致性，特别是在大型项目或复杂的继承体系中。

注意事项：

  • 要实现多态性，基类中的函数必须被声明为virtual。
  • 如果基类中的虚函数有默认实现，并且派生类希望使用这个默认实现而不是提供自己的实现，那么派生类可以不重写该函数。然而，如果派生类希望提供自己的实现，它必须使用override关键字（C++11及更高版本）来明确指示它正在重写基类中的虚函数。
  • 虚函数表（vtable）是C++实现多态性的一种机制，但它是由编译器在后台管理的，程序员通常不需要直接与之交互。
  • 虚析构函数是确保派生类对象在通过基类指针删除时能够正确析构的重要机制。如果基类析构函数不是虚函数，那么删除派生类对象时只会调用基类的析构函数，这可能会导致资源泄漏或其他未定义行为。

异常处理

C++中的异常捕获机制是一种强大的错误处理工具，它允许程序在运行时遇到错误或异常情况时，能够优雅地处理这些错误，而不是让程序崩溃或返回错误码。这种机制主要由try、throw和catch三个关键字组成。

  1. try块
    try块用于包裹可能会抛出异常的代码。在try块内部，你可以编写可能会引发错误的代码。如果这段代码在执行过程中遇到了无法处理的错误，它可以通过throw关键字抛出一个异常对象。
  2. throw语句
    throw语句用于抛出一个异常对象。这个对象可以是任何类型的，但通常是继承自std::exception的类（或其子类）的实例。throw语句会立即终止当前函数的执行，并将控制权转移到与抛出异常类型相匹配的catch块。
  3. catch块
    catch块用于捕获并处理异常。它可以紧跟在try块之后，并且可以有多个catch块来捕获不同类型的异常。每个catch块都指定了一个异常类型，并定义了当捕获到该类型异常时应执行的代码。

异常捕获的过程

  1. 异常抛出：当try块中的代码抛出异常时，程序会立即离开try块，并搜索与该异常类型相匹配的catch块。
  2. 异常传播：如果没有找到匹配的catch块，异常会继续向上层调用者传播，直到找到匹配的catch块，或者程序因为未处理的异常而终止。这个过程称为异常的传播（或堆叠展开）。
  3. 异常处理：一旦找到匹配的catch块，程序就会跳转到该块并执行其中的代码来处理异常。catch块可以通过异常对象来获取有关异常的信息，并采取相应的措施来处理它。
  4. 继续执行：处理完异常后，程序会从catch块的末尾继续执行。如果catch块是函数中的最后一个块，则程序会返回到调用该函数的代码继续执行。

断言

断言（assertion）是一种常用的调试工具或机制，用于在程序运行时验证某些条件是否为真。如果条件为假，则程序会触发一个错误，通常会导致程序终止或输出错误信息，以帮助开发者发现和修正错误。断言主要用于开发和测试阶段，以提高代码的可靠性和健壮性。
以下是对编程中断言的详细解释：

  • 作用：断言用于验证程序中某个条件是否为真。如果条件为假，则程序会报错并终止执行，从而帮助开发者定位问题所在。
  • 使用场景：断言通常用于检查那些不应该发生的错误，如逻辑错误、非法参数等。它可以帮助开发者在开发过程中及早发现和修复问题，避免问题在生产环境中才被发现。
  • 实现方式：在大多数编程语言中，都有内置的断言函数或关键字。例如，在C++中，可以使用assert宏来添加断言语句；在Python中，可以使用assert关键字来添加断言语句。
  • 注意事项：
    1.断言应该用于检查那些永远不应该发生的错误，而不是用于处理逻辑上可能会发生的错误。
    2.在生产环境中，通常会禁用断言以避免其带来的性能负担。因此，断言不应该用于处理代码的错误情况。
    3.断言的代码一般放在函数或成员函数的第一行，表达式多为函数的形参或关键状态变量。

断言是编程中一种强大的工具，它可以帮助开发者在开发过程中主动进行错误检查和预防，从而提高程序的可靠性和健壮性。

RAII

RAII指的是“资源获取就是初始化”（Resource Allocation Is Initialization），它被视作C++中最强大的编程范式之一。简单说来，它指的是，用构造函数来获取一个对象的资源，相应的，借助析构函数来释放对象的资源。
