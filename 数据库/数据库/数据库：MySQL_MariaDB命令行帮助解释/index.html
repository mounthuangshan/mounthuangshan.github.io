数据库：MySQL_MariaDB命令行帮助解释

三、Mysql/MariaDB

以下是MySQL客户端程序（如mysql）在连接到MySQL服务器时用于配置和控制客户端各种行为的命令行选项和参数的说明。以下列出了一些关键选项的解释：

一般选项：

  • -? 或 --help：显示帮助信息并退出。
  • –auto-rehash：在命令行界面中自动启用表和字段名的补全。使用--no-auto-rehash禁用。
  • -A, --no-auto-rehash：禁用自动重哈希，需要手动为表和字段补全进行重哈希。
  • –auto-vertical-output：如果查询结果超出终端宽度，则自动切换到垂直输出。
  • -B, --batch：禁用历史文件使用等交互功能，启用非交互模式（静默一些输出）。
  • –bind-address=name：指定客户端连接到服务器时应绑定的IP地址。
  • –character-sets-dir=name：指定字符集文件所在的目录。
  • –column-type-info：在查询结果中显示列类型信息。
  • -c, --comments：保留SQL查询中的注释，并将其发送到服务器。
  • -C, --compress：启用客户端和MySQL服务器之间的通信压缩。
  • –debug-info：提供MySQL客户端的调试信息。
  • -D, --database=name：指定连接到服务器时要使用的默认数据库。
  • –default-character-set=name：设置与服务器通信时的默认字符集。
  • –delimiter=name：指定用于分隔查询语句的分隔符。
  • -e, --execute=name：执行给定的SQL命令然后退出。
  • -E, --vertical：垂直打印查询结果（适用于宽结果集）。
  • -f, --force：即使发生SQL错误也强制执行。
  • –histignore=name：防止特定命令被记录到历史文件中。
  • -G, --named-commands：启用MySQL客户端的内部命令（例如，\h用于帮助）。
  • -i, --ignore-spaces：忽略函数名后的空格。
  • –init-command=name：连接到服务器时执行给定的SQL命令。
  • –local-infile：启用或禁用使用LOAD DATA LOCAL INFILE从本地文件加载数据。
  • -b, --no-beep：在出错时关闭蜂鸣声。
  • -h, --host=name：指定要连接的主机（IP或域名）。
  • -P, --port=#：指定要连接的端口号。
  • –ssl-mode=name：指定SSL连接模式（例如，DISABLED，PREFERRED）。
  • –ssl-cert=name：指定PEM格式的SSL证书。
  • –ssl-key=name：指定PEM格式的SSL密钥。
  • –ssl-ca=name：指定SSL连接的CA（证书颁发机构）文件。
  • –tls-version=name：指定要使用的TLS版本，如TLSv1.2或TLSv1.3。
  • -t, --table：以表格格式输出查询结果。
  • –tee=name：将所有输出追加到指定的文件（用于记录查询）。
  • -u, --user=name：指定登录的用户名。
  • -v, --verbose：启用详细输出，级别可递增。
  • -V, --version：输出版本信息并退出。
  • –wait：如果连接断开，则等待并重试连接。
  • –connect-timeout=#：指定等待连接建立的时间（秒）。
  • –max-allowed-packet=#：指定客户端将发送或接收的最大数据包大小。
  • –net-buffer-length=#：指定TCP/IP和套接字通信的缓冲区大小。
  • –select-limit=#：为SELECT查询返回的行数设置限制（与--safe-updates一起使用时很有用）。
  • –max-join-size=#：为连接中的行数设置限制（与--safe-updates一起使用时很有用）。
  • –show-warnings：在每个查询执行后显示警告。

SSL和加密选项：

  • –ssl-fips-mode=name：为SSL启用或禁用FIPS（联邦信息处理标准）模式（适用于OpenSSL）。
  • –ssl-session-data=name：指定用于SSL会话重用的会话数据文件。
  • –ssl-session-data-continue-on-failed-reuse：即使无法重用SSL会话数据，也允许连接成功。

安全和身份验证：

  • –connect-expired-password：指示客户端在登录时可以处理过期密码。
  • –authentication-oci-client-config-profile=name：指定OCI（Oracle云基础设施）客户端配置配置文件。
  • –plugin-authentication-kerberos-client-mode=name：指定Kerberos身份验证模式（例如，SSPI，GSSAPI）。
  • –plugin-authentication-webauthn-client-preserve-privacy：在选择WebAuthn身份验证的凭据时保持隐私。

文件和目录选项：

  • –plugin-dir=name：指定客户端插件的目录。
  • –load-data-local-dir=name：指定使用LOAD DATA LOCAL INFILE加载数据时安全的目录。

其他选项：

  • –telemetry-client：加载遥测客户端插件。
  • –syslog：将过滤后的交互命令记录到系统日志（syslog）。
  • –system-command：启用或禁用系统MySQL命令（默认情况下启用）。

这些参数允许用户配置MySQL客户端如何与服务器交互、处理SSL连接、管理输出以及执行安全任务。其中许多选项用于优化性能、故障排除连接或自定义客户端环境。
这段描述是关于MySQL（或兼容MySQL的数据库系统，如MariaDB）命令行工具如何读取默认配置文件的参数说明。MySQL命令行工具，如mysql、mysqladmin等，允许用户通过命令行参数来指定配置选项，同时也支持从配置文件中读取配置。下面是对这些参数的详细解释：

  1. 默认配置文件的读取顺序：

    • C:\WINDOWS\my.ini
    • C:\WINDOWS\my.cnf
    • C:\my.ini
    • C:\my.cnf
    • D:\Scoop\apps\mysql-lts\current\my.ini
    • D:\Scoop\apps\mysql-lts\current\my.cnf

    这些是在没有指定其他配置文件时，MySQL命令行工具会按照上述顺序尝试读取的配置文件路径。文件可以是.ini或.cnf格式，它们包含了数据库连接的默认配置选项。

  2. 读取的配置组：mysql client

    • 这表示在上述配置文件中，MySQL命令行工具将查找名为[mysql client]的配置组。这个组下的配置将应用于MySQL客户端工具。

  3. 命令行参数：

    • --print-defaults：打印程序参数列表并退出。这个选项不会实际运行MySQL命令，而是列出所有读取的配置选项和它们的值。
    • --no-defaults：不从任何选项文件中读取默认选项，除了登录文件。这意味着即使存在上述路径中的配置文件，MySQL也不会读取它们。
    • --defaults-file=#：只从指定的文件#中读取默认选项。#应替换为实际的文件路径。
    • --defaults-extra-file=#：在读取全局文件后，再读取这个文件#。这允许用户指定一个额外的配置文件，它会在全局配置文件之后被读取，从而覆盖全局配置中的相同选项。
    • --defaults-group-suffix=#：除了读取标准配置组（如[mysql client]）外，还读取带有后缀#的组。例如，如果后缀是-test，则会读取[mysql client-test]。
    • --login-path=#：从登录文件中读取这个路径#。登录文件通常包含了多个预配置的数据库连接信息，#应替换为登录路径的名称。
    • --no-login-paths：不从登录路径文件中读取登录路径。这意味着即使存在登录路径文件，MySQL也不会使用它。

这些参数提供了灵活的方式来配置MySQL命令行工具的行为，允许用户根据需要从不同的源读取配置，以及覆盖默认配置。
下面是我们看到的是MySQL命令行工具的参数列表。每个参数都有其特定的作用，用于自定义MySQL客户端的行为。让我们从头开始，逐一解析这些参数。

--auto-rehash 设置为 TRUE。根据我的记忆，auto-rehash 的作用是在每次执行查询后自动重新计算提示信息。这在某些情况下可能会影响性能，但在交互式会话中，它能提供更顺畅的体验。因此，将其设置为 TRUE 是为了提高用户的便利性。

接下来是 --auto-vertical-output 设置为 FALSE。这个参数决定了当执行一个查询时，结果是以垂直格式显示还是水平格式显示。将它设置为 FALSE 意味着默认情况下结果将以水平表格的形式显示，这是大多数用户更习惯的方式。

再来看 --bind-address，这里没有提供默认值。这个参数用于指定MySQL客户端应该绑定到的本地网络接口。如果没有指定，它通常会绑定到所有可用的接口。在生产环境中，为了安全考虑，我们可能会指定一个特定的地址。

--binary-as-hex 设置为 FALSE。这个参数决定了二进制数据是否以十六进制形式显示。将它设置为 FALSE 意味着二进制数据将按原样显示，这在处理文本数据时更为直观。

--character-sets-dir 没有默认值。这个参数指定了字符集文件的位置。如果我们在编译MySQL时指定了一个特定的字符集，或者需要使用自定义字符集，这个参数就非常有用。

--column-type-info 设置为 FALSE。如果设置为 TRUE，MySQL客户端将在结果集中包含列的数据类型信息。这在某些情况下可能有用，但通常会增加输出的复杂性。

--comments 设置为 TRUE。这个参数允许在SQL语句中包含注释。对于那些喜欢在代码中添加大量注释的人来说，这是一个方便的功能。

--compress 设置为 FALSE。这个参数决定了是否在客户端和服务器之间使用压缩连接。如果设置为 TRUE，可以减少传输数据的大小，但会增加CPU的负担。

--database 没有默认值。这个参数用于在连接时指定默认的数据库。如果没有指定，我们将连接到MySQL的默认数据库，通常是 mysql。

--default-character-set 设置为 auto。这意味着MySQL客户端将根据服务器的设置自动选择字符集。这在大多数情况下都能正常工作，但在处理特定字符集时可能需要手动指定。

--delimiter 设置为 ;。这是MySQL中分隔SQL语句的默认分隔符。如果我们将它更改为其他字符，例如 GO，那么我们可以在执行多个语句之前构建更复杂的脚本。

--enable-cleartext-plugin 设置为 FALSE。这个参数与身份验证插件有关，如果设置为 TRUE，将允许使用明文身份验证方法。出于安全考虑，通常不建议这样做。

--vertical 设置为 FALSE。与 --auto-vertical-output 类似，这个参数决定了结果集的显示格式。将它设置为 FALSE 意味着结果将以水平表格的形式显示。

--force 设置为 FALSE。如果设置为 TRUE，MySQL客户端将继续执行，即使遇到错误。这在某些情况下可能有用，但通常不建议这样做，因为错误可能是需要关注的重要问题。

--histignore 没有默认值。这个参数允许我们指定一个模式，用于忽略某些命令的执行历史。这在某些情况下可能有用，但通常不常用。

--named-commands 设置为 FALSE。如果设置为 TRUE，MySQL客户端将允许使用命名的命令，如 source 或 exit，这在某些情况下可能有用。

--ignore-spaces 设置为 FALSE。如果设置为 TRUE，MySQL将忽略命令行中的空格。这在某些情况下可能有用，但通常不常用。

--local-infile 设置为 FALSE。这个参数决定了是否允许使用 LOAD DATA LOCAL 语句。由于安全原因，通常不建议这样做，但某些应用程序可能需要它。

--no-beep 设置为 FALSE。如果设置为 TRUE，MySQL将在某些操作时禁用蜂鸣声。这在某些情况下可能有用，但通常不常用。

--host 没有默认值。这个参数指定了要连接的MySQL服务器的主机名。如果没有指定，它将尝试连接到localhost。

--dns-srv-name 没有默认值。这个参数用于指定DNS服务发现的名称，用于查找MySQL服务器的地址。这在某些情况下可能有用，但通常不常用。

--html 设置为 FALSE。如果设置为 TRUE，MySQL将输出HTML格式的结果。这在某些情况下可能有用，但通常不常用。

--xml 设置为 FALSE。与 --html 类似，这个参数决定了结果是否以XML格式输出。这在某些情况下可能有用，但通常不常用。

--line-numbers 设置为 TRUE。这个参数决定了在执行脚本时是否显示行号。这在某些情况下可能有用，但通常不常用。

--unbuffered 设置为 FALSE。如果设置为 TRUE，MySQL将禁用输出缓冲，这意味着每个查询的结果将立即显示。这在某些情况下可能有用，但通常不常用。

--column-names 设置为 TRUE。这个参数决定了在结果集中是否显示列名。如果设置为 FALSE，结果将不包含列名，这在某些情况下可能有用，但通常不常用。

--sigint-ignore 设置为 FALSE。如果设置为 TRUE，MySQL将忽略SIGINT信号，这意味着你不能使用 Ctrl+C 来中断操作。这在某些情况下可能有用，但通常不常用。

--port 设置为 0。这个参数指定了要连接的MySQL服务器的端口号。如果没有指定，它将使用默认的3306端口。

--prompt 设置为 mysql>。这个参数自定义了MySQL客户端的提示符。你可以根据个人喜好进行更改，但通常保持默认值。

--quick 设置为 FALSE。如果设置为 TRUE，MySQL将使用 --no-pager 选项，这意味着它将直接输出结果，而不是通过分页程序。这在某些情况下可能有用，但通常不常用。

--raw 设置为 FALSE。这个参数决定了结果是否以原始格式显示，没有额外的处理。这在某些情况下可能有用，但通常不常用。

--reconnect 设置为 TRUE。如果设置为 TRUE，MySQL将在连接丢失时尝试重新连接到服务器。这在某些情况下可能有用，但通常不常用。

--shared-memory-base-name 没有默认值。这个参数用于指定共享内存的基名，用于与MySQL服务器进行共享内存连接。这在某些情况下可能有用，但通常不常用。

--socket 没有默认值。这个参数指定了要使用的Unix套接字文件。如果没有指定，它将使用默认的 /var/lib/mysql/mysql.sock。

--server-public-key-path 没有默认值。这个参数用于指定服务器公钥的路径，用于某些身份验证方法。这在某些情况下可能有用，但通常不常用。

--get-server-public-key 设置为 FALSE。如果设置为 TRUE，MySQL将从服务器请求公钥。这在某些情况下可能有用，但通常不常用。

--ssl-ca 没有默认值。这个参数指定了用于验证服务器证书的CA证书文件。

--ssl-capath 没有默认值。这个参数指定了包含CA证书的目录。

--ssl-cert 没有默认值。这个参数指定了客户端证书文件。

--ssl-cipher 没有默认值。这个参数指定了要使用的SSL密码套件。

--ssl-key 没有默认值。这个参数指定了客户端私钥文件。

--ssl-crl 没有默认值。这个参数指定了证书吊销列表文件。

--ssl-crlpath 没有默认值。这个参数指定了包含CRL文件的目录。

--tls-version 没有默认值。这个参数指定了要使用的TLS版本。

--tls-ciphersuites 没有默认值。这个参数指定了要使用的TLS密码套组。

--ssl-session-data 没有默认值。这个参数指定了SSL会话数据的路径。

--ssl-session-data-continue-on-failed-reuse 设置为 FALSE。如果设置为 TRUE，MySQL将在SSL会话重用失败时继续尝试。

--tls-sni-servername 没有默认值。这个参数指定了TLS SNI服务器名。

--table 设置为 FALSE。如果设置为 TRUE，MySQL将输出结果为表格格式。这在某些情况下可能有用，但通常不常用。

--user 设置为 root。这是连接到MySQL服务器时使用的用户名。

--i-am-a-dummy 设置为 FALSE。如果设置为 TRUE，MySQL将允许执行某些可能不安全的命令。不建议在生产环境中使用。

--wait 设置为 FALSE。如果设置为 TRUE，MySQL将在连接到服务器之前等待所有先前的连接关闭。这在某些情况下可能有用，但通常不常用。

--connect-timeout 设置为 0。这个参数指定了在放弃之前等待服务器连接的时间（以秒为单位）。0表示无限等待。

--max-allowed-packet 设置为 16777216。这个参数指定了MySQL服务器可以接收的最大数据包大小。如果设置得太小，可能会导致大查询失败。

--net-buffer-length 设置为 16384。这个参数指定了用于网络传输的缓冲区大小。调整这个参数可以提高性能。

--select-limit 设置为 1000。这个参数指定了 SELECT 语句返回的最大行数。这是一个安全特性，以防止意外的大量数据传输。

--max-join-size 设置为 1000000。这个参数指定了 JOIN 操作的最大行数。这也是一个安全特性，以防止服务器过载。

--show-warnings 设置为 FALSE。如果设置为 TRUE，MySQL将显示执行操作后的任何警告。这在某些情况下可能有用，但通常不常用。

--plugin-dir 没有默认值。这个参数指定了身份验证插件的目录。

--default-auth 没有默认值。这个参数指定了默认的身份验证插件。

--binary-mode 设置为 FALSE。如果设置为 TRUE，MySQL将使用二进制模式进行输入和输出，这在某些情况下可能有用，但通常不常用。

--connect-expired-password 设置为 FALSE。如果设置为 TRUE，MySQL将允许使用过期的密码连接，但这不安全。

--compression-algorithms 没有默认值。这个参数指定了用于连接压缩的算法。

--zstd-compression-level 设置为 3。这个参数指定了使用zstd进行压缩时的压缩级别。

--load-data-local-dir 没有默认值。这个参数指定了 LOAD DATA LOCAL 命令可以从中加载文件的目录。

--authentication-oci-client-config-profile 没有默认值。这个参数指定了OCI（Oracle Cloud Infrastructure）身份验证配置文件。

--oci-config-file 没有默认值。这个参数指定了OCI配置文件的路径。

--telemetry-client 设置为 FALSE。如果设置为 TRUE，MySQL将启用遥测功能，收集使用数据并发送到MySQL服务器。这在某些情况下可能有用，但出于隐私考虑，通常不建议这样做。

--plugin-authentication-webauthn-client-preserve-privacy 设置为 FALSE。如果设置为 TRUE，MySQL将保留使用WebAuthn进行身份验证时的用户隐私。这在某些情况下可能有用，但通常不常用。

--register-factor 没有默认值。这个参数用于指定在某些身份验证方法中使用的注册因子。

--system-command 设置为 TRUE。这个参数允许执行系统命令，如 !cmd，在MySQL提示符下。这在某些情况下可能有用，但出于安全考虑，通常不建议这样做。

innochecksum 参数解释

帮助和版本参数：

  • -?, --help: 显示帮助信息并退出。
  • -I, --info: 与 -? 相同。
  • -V, --version: 显示版本信息并退出。

操作参数：

  • -v, --verbose: 详细模式（每 5 秒打印进度）。
  • -c, --count: 打印文件中的页数并退出。
  • -s, --start-page=#: 从指定的页码开始（基于 0）。
  • -e, --end-page=#: 在指定的页码结束（基于 0）。
  • -p, --page=#: 只检查指定的页码（基于 0）。
  • -C, --strict-check=name: 用户指定的严格校验和算法。
  • -n, --no-check: 忽略校验和验证。
  • -a, --allow-mismatches=#: 允许的最大校验和不匹配数。
  • -w, --write=name: 用户指定的重写校验和算法。
  • -S, --page-type-summary: 显示表空间中每种页面类型的计数。
  • -D, --page-type-dump=name: 为表空间中的每个页面转储页面类型信息。
  • -l, --log=name: 输出日志。

变量和布尔选项：

  • verbose: 是否显示详细模式，默认为 FALSE。
  • count: 是否打印文件中的页数，默认为 FALSE。
  • start-page: 开始页码，默认为 0。
  • end-page: 结束页码，默认为 0。
  • page: 指定页码，默认为 0。
  • strict-check: 严格校验和算法，默认为 crc32。
