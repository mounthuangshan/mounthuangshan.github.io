Bug记录：VS2022报错LINK _ fatal error LNK1104_ 无法打开文件“ucrtd

一、问题描述

重装系统后重新安装了VS2022，但不知道为什么在运行一个C项目时VS2022突然报了LINK : fatal error LNK1104: 无法打开文件“ucrtd.lib”以及无法打开"stdio.h"等一堆源文件的错误，在网上搜寻了一下发现没有同时遇到这种情况的。遇到无法打开"stdio.h"等一堆源文件的错误的网上大多数教程都是说重装一下windows kits以及在项目 -> 属性 -> C/C++ ->常规 中的使用Windows运行时扩展，选择是然后再取消的玄学做法；而遇到LINK : fatal error LNK1104: 无法打开文件“ucrtd.lib” 报错的解决方法是在磁盘中找到ucrtd.lib文件，然后在项目属性页 -> VC++目录 -> 库目录下添加进去文件路径，这种方法虽然理论上没问题，但治标不治本，遂记录一下解决方法。

二、问题排查与解决

作者在项目的配置属性页面->VC++目录->包含目录中发现了默认路径指向的位置并没有诸如stdio.h等一系列源文件，或者说包含的不全，如下图：
